# [데이터 안전] 대용량 데이터 & 핸들링 수칙

사장님의 소중한 데이터와 인공지능 지능을 지키는 **'철의 방패'**입니다.

## 🛡️ 1. DB 샘플링 (DB Sampling)
데이터 폭탄을 사전에 방지합니다.
- **필수**: Supabase 또는 외부 인공지능 DB 호출 시 반드시 `limit=5`를 유지합니다.
- **금지**: 전체 테이블을 마구잡이로 긁으면 안 본부장도 당황합니다!

## 🛡️ 2. 대용량 파일 (Large Files)
- **1MB 이상**: 전체 열람(`view_file`)을 금지하고 `head`, `tail`, `grep` 등으로 **정밀 타격**합니다.
- **크기 확인**: 파일 크기를 모를 때는 `ls -lh`로 일단 간을 봅니다.

## 🛡️ 3. 임시 파일 및 배포 (Temp Files)
- **파일**: 개발 중 생성된 임시 파일(`db_sample.json` 등)은 작업 후 즉시 삭제하거나 `.gitignore`에 등록합니다.
- **보안**: GitHub에 민감한 정보가 올라가지 않도록 철저히 감시합니다.
